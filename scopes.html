AngularJS Scope

  • The scope is the binding part between the HTML(view) and the JavaScript(controller)
  • The scope is an object with the available properties and methods
  • The scope is availbale for both view and controller
  • When we make a controller in angularjs application, we pass a $scope object as an argument
  • when adding properties to $scope object in the controller, view(HTML) gets access to these properties

Understanding the scope

  • Angularjs application consists of
  • the template(view), which is the HTML
  • the scope(model), which is the data available in the current view
  • the controller, which is the JavaScript function that makes/changes/removes/control the data

Then the scope is model.
The scope is a Javascript object with the properties and methods, which are available for both view and controller

rootScope

  • All applications have $rootScope which is the scope created on the HTML element that contains ng-app directive
  • The $rootScope is available in the entire application
  • if a variable has the same name for the current scope and $rootScope, the application uses the one in the current scope

scope hierarchy

Every application developed in AngularJS has one $rootScope object and many child $scope objects. when a new scope is created that is added to the parent scope. This results the creation of hierarhy structure like DOM structure.
