AngularJS

Introduction

AngularJS is a JavaScript framework. It can be added to an HTML page with a script tag.
AngularJS extends HTML attributes with ng-directives and binds data to HTML with expressions or ng-bind directive.

AngularJS directives

ng-app directive
ng-app directive defines the root element of an AngularJS application.
ng-app directive will auto-bootstrap(automatically initializes) the application when a web page is loaded.
ng-model directive
ng-model directive binds the values of HTML controls(input, textarea, select) to application data.
ng-model directive can also:
  • provide type validation to application data(number, email, required)
  • provide status for application data(invalid, dirty, touched, error)
  • provide css classes for HTML elements
  • provide HTML elements to HTML forms
ng-bind directive
ng-bind directive binds application data to HTML view
ng-init directive
ng-init directive initializes angularjs application variables
ng-controller directives
ng-controller directive defines the application controller

AngularJS Module

An AngularJS module defines an application.A module is a container for different parts of an application
and a module is a container for application controllers. Controllers always belongs to a module.

creating a module

A module is created by using angular.module

AngularJS Controllers

  • AngularJS controller controls the data of AngularJS application.
  • AngularJS controllers are regular JavaScript objects.
  • AngularJS applications are controlled by controllers.
  • ng-controller directive defines the application controller.
  • A controller is a JavaScript object, created by a standard JavaScript object constructor.

AngularJS Data Binding

  • Data binding in AngularJS is synchronization between the model and the view.
  • AngularJS application usually have a data model. A data model is a collection of data available for the application
  • The HTML container where the AngularJS application is displayed is called a view.

AngularJS Scope

The scope is the binding part between the HTML(view) and JavaScript(Controller).
The scope is an object with the availabe properties and methods.
The scope is available for both the view and controllers.

How to use the scope ?

When we make a controller in AngualrJS, we pass the $scope object as an argument.
When adding properties to $scope object in the controller, the HTML(view) gets access to these properties.
If we make changes in the view, the model and controller will be updated.

Understanding the scope

  • View, which is the HTML
  • Model, which is the data available for the current view
  • Controller, which is the JavaScript function that makes/changes/removes/controls the data.

AngularJS rootScope

All applications have $rootScope which is a scope created in an HTML element that contains ng-app directive.
The $rootScope is available in the entire application.
If a variable has the same name in the current scope and $rootScope, the application uses the one in the current scope.
